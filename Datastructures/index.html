Data structures || JS

JS has been used more and more as a server-side computer programming language.

Data structures is systematic way to organize data in order to use it efficiently.

In this file i focus on implementation of data structures with javascript.

What are "Data structures"?

Data Structures allow you to manage data.

Ex: Arrays, Objects, Maps, Sets...

Arrays: [1,2,3]
objects: { name: 'Max', age: 31 }
Maps: new Map() map.set('a','b')
Sets: new Set() set.add(1)

Different Tasks Require Different Data Structures.

  • Ordered List of Data, Duplicates Allowed.
    • Array
      • [1,2,5,3]
  • Unordered List of Data, No Duplicates wanted.
    • Set
      • new Set()
      set.add('pizza')
  • Key-value Pairs of unordered Data.
    • Object
      • {name: 'Max',age:31}
  • Key-value pairs of ordered, iterable Data.
    • Map
      • new Map()
      map.set('local','Germany')

Arrays

[1,3,6,2]

  • Insertion order is kept.
  • Element access vis index.
  • Iterable (= you can use the for-of loop).
  • Size(length) adjusts dynamically.
  • Duplicate values are allowed.
  • Deletion and finding elements can require "extra work".

Sets

Sets ia all about lists of data.

new set ()
set.add('Pizza')
set.add('burger')
set.add('Pizza') // not added

  • Insertion Order is not stored/ memorized.
  • Element access and extraction via method.
  • Iterable (= you can use the for-of loop).
  • Size(length) adjusts dynamically.
  • Duplicate values are not allowed(i.e. unique values only)
  • Deletion and finding elements is trivial and fast.

Arrays


You can always use arrays

Must-use if order matters and/or duplicates are wanted.

Sets


Only usable if order does not matter and you only need unique values.

Can simplify data access (finding,deletion) compared to arrays.

Objects

{ name: 'Max', age: 31,
greet(){
console.log('Hi, I am ' +this.name);
} }

  • Unordered key-value pairs of data.
  • Element access via key (property name)
  • Not iterable (only with for-in)
  • Keys are unique, values are not.
  • Keys have to be strings, numbers or symbols
  • Can store data & "functionality" (methods)

Maps

new Map()
map.set('name','Max')
map.set(true,tru) //Boolean Key

  • Ordered key-value pairs of data.
  • Element access via Key.
  • Iterable (=you can use the for-of loop).
  • Keys are unique,values are not.
  • Keys can be anything (incl, reference values like arrays).
  • Pure data storage, optimized for data access
